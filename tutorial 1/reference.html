Next Steps and Troubleshooting

Keep this page as your operating manual. Building software is iterative, and this gives you a stable process to improve your app over time.

How to Make Changes Safely

  1. Describe your desired change to Cursor.
  2. Run locally with npm run dev.
  3. Test the specific flow you changed.
  4. Push to GitHub and confirm Vercel deploy succeeds.

Feature Ideas for Your Negotiation Platform

  • Add editable negotiation strategy presets (conservative, balanced, aggressive).
  • Add a comparison view showing student preference score vs agent score per offer.
  • Add negotiation transcripts so students can review past offer rounds.
  • Add product category filtering (laptop, phone, accessories) for easier selection.

Troubleshooting FAQ

  • Open browser Developer Tools: right click -> Inspect -> Console.
  • Copy the red error message.
  • Ask Cursor to fix it and explain root cause simply.
  • Run npm install first to ensure dependencies are present.
  • Check the first error line, not only the final line.
  • Paste full error into Cursor Composer for guided fixes.
  • Check Vercel build logs for exact failure file and command.
  • Verify NEXT_PUBLIC_INSTANT_APP_ID is set in Vercel project settings.
  • Confirm all files are pushed to GitHub main branch.
  • Check App ID accuracy in local and Vercel environments.
  • Use a real email address and check spam folder.
  • Confirm auth setup in InstantDB dashboard.
  • Verify push to GitHub actually completed.
  • Wait 1-2 minutes for Vercel to rebuild.
  • Hard refresh your browser (Cmd+Shift+R or Ctrl+Shift+R).
  • Create a fresh folder and rerun the build prompt.
  • Use your same InstantDB app or create a new one for a clean dataset.
  • This is normal. Restarting is often faster than patching many broken pieces.
