Welcome - You are building a real web app

You are building your own E-Commerce Negotiation Platform, where users and a computer agent exchange offers. You do not need to code from scratch; you will guide AI clearly and let it generate the code.

What you are building

Your app lets an Admin configure a negotiation topic (for example, buying a laptop), define features/options, and set preference weights. Then Users negotiate with an agent by exchanging offers and seeing utility values.

Simple finished-app picture

  • Admin creates templates and agent preferences
  • User sets profile + preferences and starts a session
  • User and agent exchange offers until accepted or ended

Understanding dynamic web apps

Website vs web app

A website is mostly read-only information. A web app is interactive: users log in, save data, and get personalized behavior. Your project is a web app because each user account stores its own negotiation sessions.

Client-server model (restaurant analogy)

Think of a restaurant. The dining room is what customers see. The kitchen is where food is prepared and stored. In your app, the frontend is the dining room, backend is the kitchen, and API is the waiter carrying requests.

Why both frontend and backend matter

  • Persistence: data survives when browser tab closes
  • Multiple users: shared central data with private user ownership
  • Security: permissions must be enforced outside the browser
