Next steps and troubleshooting

This is your ongoing guide for improving and debugging your app after launch.

How to make changes safely

  1. Describe changes in Cursor Composer
  2. Test with npm run dev
  3. Push to GitHub
  4. Let Vercel auto-redeploy

Feature ideas for your next version

  • Add edit/delete for templates and feature options
  • Add negotiation analytics (acceptance rate, average utility)
  • Add reusable saved preference profiles
  • Add filters for sessions by topic and outcome

Troubleshooting FAQ

Open browser console, copy red errors, and ask Cursor to fix them.

Run npm install, then paste full error text into Cursor if needed.

Check Vercel logs and confirm NEXT_PUBLIC_INSTANT_APP_ID is set.

Verify App ID, email spelling, and spam folder for verification code.

Make sure latest commit is on GitHub, wait 1-2 minutes, then hard refresh.
