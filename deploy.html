Deploy your app to the internet

Deployment means moving your app from your laptop to a cloud server that stays on all day. Vercel handles this automatically once your code is on GitHub.

Push your project to GitHub

  1. Create a new public repository on GitHub
  2. Open your project in Cursor
  3. Ask Cursor: "Help me push this project to GitHub. Here is my repository URL: ..."

Why this step exists: Vercel does not read from your laptop directly. It reads code from GitHub, then builds it.

Deploy with Vercel

  1. Sign into Vercel with GitHub
  2. Click Add New Project and select your repository
  3. Add environment variable before deploy:
Name: NEXT_PUBLIC_INSTANT_APP_ID
Value: your-real-app-id-from-instantdb
  1. Click Deploy
  2. Wait for build completion and open the provided app URL

Why environment variables? They are configuration values passed at build/runtime without hardcoding sensitive or changeable values into files. Your InstantDB App ID is public-safe, but still better managed as config.

Post-deploy checklist

  • Can you log in with Magic Code?
  • Can an Admin create template data?
  • Can a User start a session and exchange offers?
  • Does page refresh keep saved data?

If something fails in production, copy Vercel build logs and ask Cursor to fix based on those logs.

Deploy Your App to Vercel

Deployment means moving your app from your laptop to an always-on server. After this step, anyone with your URL can use your negotiation app.

1) Create a GitHub Repository

  1. Sign in to GitHub.
  2. Click + then New repository.
  3. Name it something like negotiation-platform.
  4. Set visibility to Public.
  5. Do not initialize with README or .gitignore.

2) Push Code from Cursor to GitHub

If Git feels new, ask Cursor to guide each command. Paste your repo URL in the prompt.

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin YOUR_GITHUB_REPO_URL
git push -u origin main

Why this matters: GitHub is your source of truth. Vercel watches your repository and redeploys whenever code changes.

3) Deploy in Vercel

  1. Open vercel.com and sign in.
  2. Click Add New Project.
  3. Select your GitHub repository.
  4. Before clicking Deploy, add environment variable:
Name: NEXT_PUBLIC_INSTANT_APP_ID
Value: your real InstantDB App ID
  1. Click Deploy.
  2. Wait for build success and open your public app URL.

Environment Variables in Plain Language

Environment variables are named boxes for configuration values. They keep deployment settings separate from code files. In this app, your InstantDB App ID is public-safe, so using NEXT_PUBLIC_ is correct.

4) Automatic Updates

After first deployment, each push to GitHub triggers a new Vercel deploy automatically. This means your update process becomes: edit in Cursor, test locally, push, then verify live URL.
